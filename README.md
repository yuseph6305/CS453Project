# CS453 MPI Distributed Algorithms

This project takes graphs generated by NetGameSim and runs two distributed algorithms using MPI: FloodMax leader election and Dijkstra shortest paths. Each MPI rank owns a subset of graph nodes, and ranks communicate via message passing for cross-rank edges.

## Dependencies

```bash
brew install open-mpi cmake python3
```

## Step 1 - Graph files

The generated graph DOT files are already in outputs/. To regenerate them, use NetGameSim:

```bash
cd NetGameSim
java -Xms2G -Xmx8G -jar target/scala-3.2.2/netmodelsim.jar MyGraph
```

## Step 2 - Export graph to JSON

```bash
python3 tools/graph_export/graph_export.py outputs/MyFirstGraph.ngs.dot outputs/graph.json --seed 42
```

## Step 3 - Partition graph across ranks

```bash
python3 tools/partition/partition.py outputs/graph.json --ranks 4 --strategy contiguous --out outputs/part.json
```

## Step 4 - Build the MPI program

```bash
cmake -S mpi_runtime -B build
cmake --build build
```

## Step 5 - Run leader election

```bash
mpirun -n 4 ./build/ngs_mpi --graph outputs/graph.json --part outputs/part.json --algo leader --rounds 200
```

## Step 6 - Run Dijkstra

```bash
mpirun -n 4 ./build/ngs_mpi --graph outputs/graph.json --part outputs/part.json --algo dijkstra --source 0
```

## Correctness Assumptions

- **Positive edge weights** — Dijkstra requires all weights to be greater than 0. The export tool enforces this with `max(1, weight)`.
- **Connected graph** — Both algorithms require a fully connected graph. The export tool automatically bridges any disconnected components.
- **Unique node IDs** — FloodMax requires unique integer node IDs in range 0 to N-1. The export tool remaps all IDs to ensure this.

## Project Layout

```
tools/partition/       Splits graph nodes across MPI ranks
mpi_runtime/src/       Main MPI program in C (leader election + Dijkstra)
mpi_runtime/include/   Shared header with graph and partition structs
tests/                 Unit and correctness tests
outputs/               Generated graphs, partitions, and DOT files
configs/               Example run configurations
experiments/           Scripts to reproduce all experiments
REPORT.md              Experiment writeup and analysis
```
